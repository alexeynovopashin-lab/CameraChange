# Документы проекта Camerateka

Иерархия документов — от смысла к плану:

1. **[Vision](vision.md)** — зачем проект существует. Меняется реже всего.
2. **[Конституция](constitution.md)** — по каким правилам принимаются решения. Неприкосновенные принципы.
3. **[Свод проекта](svod.md)** — что именно строим: архитектура, разделы, механики.

Рабочие документы:

- **[Дорожная карта](roadmap.md)** — что сделано, что в работе, что впереди.
- **[Журнал прогресса](progress.md)** — хронология значимых изменений.

Единый источник истины — эти файлы в репозитории. Word-версии считаются черновиками и не являются действующей редакцией.

Полная история изменений — в git (`git log`).
