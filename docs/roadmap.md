# Дорожная карта

Живой документ. Отражает [порядок развития из Свода](svod.md#12-порядок-развития):
подбор → камератека → скоринг → сообщество → база знаний → маркетплейс → экосистема.

Критерий добавления любой функции — [четыре вопроса из Конституции](constitution.md):
делает ли фотографию понятнее, усиливает ли экосистему, помогает ли принять решение,
будет ли полезно через десять лет.

## Пересмотр опросника (план)

Порядок: **сначала данные, потом формулировки** — иначе строим на песке.

1. Дозаполнить feature-флаги (см. выше).
2. Убрать мёртвые вопросы (`type: ignore`) или дать им вес: б/у, «кто поможет»,
   «что уже есть». Особенно оживить p7/p8 (текущая система → «остаться в системе»).
3. Разделить вопрос об автофокусе: «портрет» и «движение/дети/животные» — разные сценарии.
4. Убрать дубли (eye AF и LOG считаются дважды в p4/p5 и p10).
5. Переписать спец-вопросы в задачи (Vision: человек думает задачами, не характеристиками):
   «важны ли мегапиксели» → «будете печатать крупно / кадрировать».
6. Добавить инженерную ветку (стоп-моушен / предметка).
